물리 개념 ↔ 엔트리 구현 분리 (외부 리뷰 반영)

작성일 2026-05-29 대상: 시리즈 전 편의 콘텐츠 구조·표현 정확성

외부 콘텐츠 리뷰(review_20260529.md)를 반영해, “배울 물리 개념”과 “엔트리로 흉내 낸 구현”을 구조적으로 분리하는 규칙을 도입한다. 규칙은 가이드 문서에 먼저 기록하고, 콘텐츠는 3편에 파일럿 적용했다.

TL;DR
  • 핵심 도입: 🔬 물리 개념(개념 읽기)과 🧩 엔트리 구현(따라 만들기)을 별도 섹션으로 가른다. 새 프리미티브 .goals·.tip.real·.fix + 표준 편 구조(design-guide D-4).
  • 정확성 보정: 관성=‘성질’(힘 아님), vy는 +y(위쪽), 45° 최대 사거리는 조건부 등 — 영구 참조표를 내용 가이드 A-3에 둔다.
  • 정보 먼저, 적용은 점진: 모든 규칙을 가이드에 기록 → 3편만 파일럿. 1·2·4·5편 확대는 후속.
  • 제외: 교사/부모용 자료·수업 운영 메타 — 독자는 학습자 본인이므로 반영하지 않음.

배경

review_20260529.md는 시리즈의 방향성을 호평하면서 네 가지 보완점을 제안했다 — (1) 물리 정확 표현과 게임 흉내의 구분, (2) 개념 읽기와 따라 만들기의 분리, (3) 난이도 단계화, (4) 오류 해결 코너. 여기에 사용자가 세 가지 결정을 더했다.

채택 안 — 분리 장치

새 프리미티브 (docs/assets/styles.css)
  • .goals — 편 상단 학습 목표 박스. 🔬 물리 개념 / 🧩 엔트리 구현 2칼럼(초록/파랑). 수업 운영용 메타는 넣지 않는다.
  • .tip.real — 「🌍 현실 vs 🎮 게임」 콜아웃(초록 보더). 비유가 현실과 어긋나기 쉬운 지점에 사용.
  • .fix — 「🔧 안 될 때 확인!」 오류 해결 박스(주황 보더). 따라 하다 막히는 지점.
  • 기존 .tip.bonus(보라)는 ⭐ 도전(고난도 분리)으로 재활용.

표준 편 구조(개념↔구현 분리형) — 정본은 design-guide.html D-4:

  1. 인트로 hook — 앞 편 경험·불편함에서 “왜?”를 체험
  2. 🎯 학습 목표(.goals) — 🔬물리 / 🧩구현
  3. 🖼️ 미리보기 — 오늘 만들 장면(데모)
  4. 🔬 물리 개념(개념 읽기) — 현실 개념 + 「현실 vs 게임」
  5. 🧩 엔트리로 만들기(따라 만들기) — 오브젝트·변수 → 블록 → 실행
  6. 🔧 안 될 때 확인!(.fix)
  7. 🧪 숫자 바꿔 실험 → 🔁 코드에서 찾기 → ⭐ 도전 → ✅ 마무리

정확성 원칙 — 현실 물리 vs 게임 흉내

리뷰가 짚은 “쉽게 쓰려다 어긋난” 표현을 영구 참조표로 남겼다. 정본은 authoring-editing-guide.html A-3.

개념흔한 오류정확한 표현
관성“유지하려는 힘”힘이 아니라 운동 상태를 유지하려는 성질
+0.8 / ×0.85“입력 힘 / 마찰”로 단정진짜 힘이 아니라 게임용 간단 규칙임을 덧붙임
마찰 방향“마찰은 x에만”“이번 게임에서는 x에만” (현실엔 y 저항도)
종단속도현실 종단속도 = 게임 낙하 제한둘을 구분(현실 vs 제한값)
최대 사거리“45°가 무조건 최대”“평평한 바닥·같은 높이·공기 저항 없음”일 때
발사힘↔사거리“2배 → 2배”사거리 ∝ 속도² → “훨씬 멀리”
좌표계 부호vy 부호가 페이지 안에서 엇갈림엔트리는 위쪽 +y → 위로 쏘면 vy는 +값

3편 재구성 (파일럿)

이전 — 주제별 혼합

관성(개념+x코드) → 마찰(개념) → 중력(개념+y코드) → 서브스텝 → 코요테

※ 마찰 코드(×0.85)가 마찰 개념 설명보다 먼저 등장.

이후 — 개념↔구현 분리

🎯목표 → 🔬관성·🔬마찰·🔬중력(개념) → 🧩엔트리로 만들기(x·y 코드 통합) → 🔧안 될 때 → ⭐도전(서브스텝·코요테)

※ 개념을 먼저 한 덩어리로, 코드는 한 곳에 모음.

반영하지 않은 것 (사용자 지시)
  • 교사용 자료 — 수업 지도안·활동지 PDF·완성 .ent 예제·빈 프로젝트·평가 루브릭·발표 질문.
  • 수업 운영용 메타 — 예상 시간·대상 학년(학습 목표 박스는 물리/구현 2분할만).
  • 선수 개념·결과물 필드 — 박스에 넣지 않음(“오늘 만들 장면”은 별도 미리보기 섹션).
