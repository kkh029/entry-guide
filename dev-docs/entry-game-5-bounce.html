5편 「튕기기와 탄성」 추가

작성일 2026-05-29 대상 docs/entry-game-5-bounce.html PR 생성 예정

탄성 충돌(튕김·반사·탄성계수)을 가르치는 5편을 시리즈 마지막 편으로 추가하고, 통통 튀는 공 캔버스 데모를 새로 만들었다.

TL;DR
  • 시리즈에 없던 개념(튕김/반사/탄성)을 5편으로 신규 추가 — 4편의 "쿵 멈춤"을 "통통 튕김"으로 잇는 브리지.
  • 핵심 규칙: 부딪히는 순간 속도에 음수를 곱한다(방향 뒤집기 × 힘 줄이기). 3편 마찰(×0.85, 매 프레임)과 같은 원리·다른 시점으로 연결.
  • 인터랙티브 데모: 탄성계수 슬라이더(0.5~1.0)로 통통 정도를 체감. 헤드리스 검증에서 6회 튕기고 ~185프레임에 떨림 없이 정착.
  • 새 액센트 --c5(틸 #3aa6a0) 토큰 + .c-teal 추가. 범위는 튕김·탄성 코어만(공끼리 충돌/당구는 6편 후보로 보류).

개요

기존 시리즈는 1편(이동·충돌) → 2편(속도·가속도) → 3편(관성·마찰) → 4편(포물선·발사)로, 4편이 마지막이었다. 탐색 결과 어떤 편도 "튕김/반사/탄성"을 다루지 않았다 — 1편 충돌은 밀어내기(push-out)뿐이고, 4편 발사체는 바닥에 닿으면 그냥 멈춘다. 게임에서 매우 흔한 "튕기는 공"을 가르칠 자리가 비어 있었고, 이를 5편으로 채웠다.

제목·범위는 사용자 결정으로 확정했다: 제목 「튕기기와 탄성 — 통통 튀는 공 만들기」, 범위는 튕김·탄성 코어만(공끼리 충돌은 제외).

이슈 — 채워지지 않은 빈틈

충돌 후 반사/튕김을 다루는 편이 없었다. 1편은 충돌을 "겹침 감지 + 밀어내기"로만 설명하고, 4편 발사체는 착지하면 정지한다. "공이 바닥에서 튀어 오른다"는 직관적이고 게임에 흔한 동작을 학습자가 만들 방법이 시리즈에 없었다.

해결 — 5편 신규 + 시리즈 배선
  • 4개 코어 섹션: ① 데모 체험 → ② 튕김=방향 뒤집기(바닥 vy, 벽 vx) → ③ 탄성계수(튈 때마다 잃는 힘, ×0.7) → ④ 엔트리 구현. 마지막에 "마찰 vs 탄성" 정리 + 코드 찾기 퀴즈.
  • 통통 공 데모: 중력 낙하 + 바닥/벽 반사 + 탄성계수 슬라이더. 2편/4편 데모와 동일한 self-contained IIFE + requestAnimationFrame 패턴.
  • 3편 마찰과 명시적 연결: 둘 다 "속도에 1보다 작은 수를 곱한다"는 같은 아이디어, 단 마찰은 매 프레임·탄성은 충돌 순간.
  • 시리즈 배선: 4편 종료 CTA를 5편 브리지로 교체, index에 ep-5 카드·5스텝 path 추가, 기존 "5편 후보" 블록은 "6편 후보"로 전환.

섹션제목가르치는 것

2튕김이란? — 속도 방향이 뒤집혀요바닥→vy 부호 뒤집기, 벽→vx 부호 뒤집기 (axis-split 비교)

4엔트리로 튕기는 공 만들기중력 낙하 코드 + y속도 = y속도 × -0.7 한 줄, 정착 처리

데모 물리

캔버스 좌표는 +y가 아래다(낙하 = vy 증가). 매 프레임 중력으로 vy를 키우고 위치를 적분한 뒤, 바닥/벽에 닿으면 해당 축 속도의 부호를 뒤집으며 탄성계수를 곱한다. 바닥에서 vy가 충분히 작아지면 0으로 고정해 미세 떨림을 없애고, 그 뒤 수평 속도에 마찰을 줘 굴러 멈추게 한다(3편 마찰 호출).

상수값의미
gravity0.5매 프레임 vy 증가량
restitution0.7 (슬라이더 0.5~1.0)바닥 튕김 시 남는 속도 비율
wallRest0.9좌우 벽 반사 시 남는 속도 비율
settleVy1.2이보다 작은 vy면 0으로 고정(정착)
시작값x=W/2, y=60, vx=2.2, vy=0상단 중앙에서 살짝 옆으로
 
function loop(){
  if (!running) return;
  vy += gravity;                 // 중력
  x += vx; y += vy;              // 위치 적분
  let settled = false;
  if (y + r > groundY){          // 바닥 충돌
    y = groundY - r;
    if (Math.abs(vy) < settleVy){ vy = 0; settled = true; }   // 정착
    else { vy = -vy * restitution; bounceCount++; }           // 튕김: 부호 뒤집기 + 힘 줄이기
  }
  if (settled){                  // 바닥에서 굴러 멈춤 (3편 마찰)
    vx *= 0.94;
    if (Math.abs(vx) < 0.12) vx = 0;
  }
  if (x - r < 0){ x = r;     vx = -vx * wallRest; }   // 좌측 벽 반사
  if (x + r > W){ x = W - r; vx = -vx * wallRest; }   // 우측 벽 반사
  trail.push({ x, y }); if (trail.length > 55) trail.shift();
  render();
  requestAnimationFrame(loop);
}
HUD 표기 — "위가 +"

HUD의 y속도는 (-vy)로 표기한다(4편 관습). 낙하 중엔 음수, 튕긴 직후엔 양수로 보여, 튕기는 순간 부호가 뒤집히는 것이 숫자로 그대로 드러난다.

엔트리 블록 표현

튕김 핵심 줄은 계산 카테고리 곱하기 블록을 자료 "정하기" 블록에 중첩해 표현한다(3편 마찰 ×0.85와 동일 패턴).

계속 반복하기:
  y속도에 -0.8 만큼 더하기            ← 중력
  y 좌표를 (y속도) 만큼 바꾸기
  만일 〈바닥에 닿았는가?〉 이라면:
    y속도를 (y속도 × -0.7) 로 정하기   ← 튕김: 방향 뒤집기 + 힘 줄이기
렌더러 한계 — 중첩 reporter는 텍스트 폴백

B-mini 렌더러는 boolean/calc 안의 깊은 중첩(예: y속도의 절댓값 < 1)을 plain-text로만 그린다. 그래서 정착 조건 같은 부등식은 3편이 쓰는 텍스트 폴백 방식({"type":"text","value":"‘y속도’의 절댓값 < 1"})으로 작성했다. 단순 변수 × 숫자 곱하기는 reporter로 정상 렌더된다.

설계 결정

범위 — 코어만

"탄성 충돌"은 (a) 튕김/반사와 (b) 공끼리 운동량 교환(당구/핀볼)으로 나뉜다. 한 개념을 깔끔하게 전달하고 3편 마찰과 직접 잇기 위해 (a) 코어만 채택했다. (b)는 6편 후보로 보류.

마지막 편 CTA

5편이 새 마지막 편이므로 CTA는 "시리즈를 모두 마쳤어요!" 완주 변형으로 두되, new-page-guide.html 권고대로 목차(index)로 돌아가는 링크를 달아 막다른 길을 피했다.
