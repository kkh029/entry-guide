7편 「운동량과 충돌」 추가

작성일 2026-05-31 대상 docs/entry-game-7-collision.html PR 생성 예정

운동량(질량 × 속도)과 공끼리 충돌(속도 교환)을 가르치는 7편을 시리즈 새 마지막 편으로 추가하고, 큐볼 질량에 따라 충돌 결과가 달라지는 당구 캔버스 데모(1D 탄성 충돌)를 만들었다.

TL;DR
  • 사용자가 7편 주제로 「운동량·충돌」(리뷰 2순위, 물리 계속)을 선택 — "점수·게임 규칙"(로드맵 7편) 대신 물리 학습 흐름 유지.
  • 핵심 개념: 운동량 = 질량 × 속도(6편 질량 연결), 충돌 = 속도 주고받기(같은 무게면 맞바꾸기), 에너지 손실(5편 탄성 연결).
  • 엔트리 핵심 = ‘임시속도’ 변수로 두 공의 속도를 안전하게 스왑(고전적 swap 패턴). 공A/공B 두 오브젝트 스크립트.
  • 당구 데모: 큐볼 질량 슬라이더(1~10, 흰 공=4 고정)로 1D 탄성 충돌 체감 — 같은 무게 큐볼 정지, 무거우면 계속 전진, 가벼우면 튕겨 되돌아옴.
  • 새 액센트 --c7(와인·버건디 #9a3b52) + .c-wine. 7편이 새 마지막 편 → 6편 CTA를 7편으로 잇고(완주 문구 제거), index "7편 후보"→"8편 후보"로 전환.

개요

6편(힘·질량)까지 추가된 시리즈에서, 5편(튕김·탄성)은 공 하나가 벽/바닥에 부딪혀 튀는 1축 반사만 다뤘다. 공끼리 부딪히는 충돌(운동량 전달)은 5편의 후속 TODO("공끼리 충돌 — 당구/핀볼")로 보류돼 있었다. 7편 주제를 정할 때 두 갈래가 있었다 — (a) 리뷰 로드맵의 7편 「점수·게임 규칙」, (b) 리뷰 추가 주제 2순위 「운동량·충돌」(물리 계속). 사용자가 (b) 물리 계속을 선택해, 시리즈의 "물리를 배운다" 정체성을 유지했다.

제목·범위·디자인은 사용자 결정으로 확정: 제목 「운동량과 충돌 — 당구공처럼 부딪혀 보자!」, 인터랙티브 당구 데모 포함, 액센트 --c7 = #9a3b52(와인·버건디).

이슈 — 채워지지 않은 빈틈

공이 다른 공에 부딪히는 충돌(운동량 전달)을 다루는 편이 없었다. 5편은 벽·바닥 반사(혼자 튕김)만, 6편은 힘으로 미는 것만 다뤘다. "당구처럼 친 공이 멈추고 맞은 공이 굴러간다"를 만들 자리가 비어 있었다.

해결 — 7편 신규 + 시리즈 배선
  • 개념↔구현 분리 구조: .goals 🔬/🧩 → 🔬운동량 → 🔬충돌(속도 교환) → 🧩엔트리 만들기(속도 스왑) → 🔧안 될 때 확인 → ⭐도전(벽돌깨기) → 🔍코드 찾기 → ✅마무리.
  • 당구 데모: 큐볼/흰 공 1D 탄성 충돌. 큐볼 질량 슬라이더로 결과 변화 체감. 5편/6편 .try-demo 패턴 재사용.
  • 6편 질량·5편 탄성과 명시적 연결: 운동량=질량×속도(6편), 에너지 손실 옵션 ×0.9(5편·3편 "1보다 작은 수 곱하기").
  • 시리즈 배선: 6편 종료 CTA를 7편 브리지로 교체(완주 문구 제거), index에 ep-7 카드·7스텝 path 추가, "7편 후보(점수·게임 규칙)"→"8편 후보(점수·게임 규칙/에너지/경사면)" 전환.

데모 물리 — 1D 탄성 충돌

두 공이 한 직선 위에서 굴러 부딪힌다. 큐볼이 흰 공의 왼쪽 가장자리에 닿으면(x1+r ≥ x2−r, 그리고 다가가는 중 v1>v2) 1D 완전 탄성 충돌 공식으로 두 속도를 갱신한다. 큐볼 질량 슬라이더(흰 공 질량=4 고정)에 따라 세 가지 결과가 나온다.

큐볼 질량 m1충돌 후 큐볼 v1'충돌 후 흰 공 v2'현상
4 (같음)0.04.2큐볼 정지, 흰 공이 그대로 출발 (속도 맞바꾸기)
10 (무거움)1.86.0큐볼 계속 전진, 흰 공 더 빠르게
1 (가벼움)−2.521.68큐볼 튕겨 되돌아옴, 흰 공 천천히
 
// 발사 속도 LAUNCH=4.2, 흰 공 질량 m2=4 고정, 큐볼 질량 m1=슬라이더(1~10)
if (!collided && (x1 + r) >= (x2 - r) && v1 > v2){
  x1 = x2 - 2 * r;                                     // 겹침 보정
  const u1 = v1, u2 = v2;
  v1 = ((m1 - m2) * u1 + 2 * m2 * u2) / (m1 + m2);     // 1D 탄성 충돌 (큐볼)
  v2 = ((m2 - m1) * u2 + 2 * m1 * u1) / (m1 + m2);     // (흰 공)
  collided = true;
}
HUD + 안내문 — "속도를 주고받음"을 보여줌

좌/우 HUD에 큐볼 속도·흰 공 속도를 실시간 표시하고, 충돌 직후 캔버스 상단에 질량 비교 결과 문구("같은 무게! 큐볼이 멈추고 속도를 넘겨줬어요" 등)를 띄운다. 본문 설명은 "속도를 주고받는다" 수준으로 유지하고(리뷰 권장), 정확한 공식은 데모 내부 + ⭐보너스에만 둔다.

엔트리 블록 표현 — 속도 스왑

충돌의 핵심은 두 변수의 값 맞바꾸기(swap)다. 임시 변수 없이 바로 대입하면 첫 줄에서 값이 덮어써지므로, ‘임시속도’에 잠깐 저장하는 고전적 swap 패턴을 가르친다.

[공A · 큐볼]
시작하기 버튼을 클릭했을 때
A속도를 4 로 정하기                       ← 오른쪽으로 발사
계속 반복하기:
  x 좌표를 (A속도) 만큼 바꾸기
  만일 〈공B에 닿았는가?〉 이라면:
    임시속도를 (A속도) 로 정하기            ① A속도 잠깐 저장
    A속도를 (B속도) 로 정하기              ② A가 B의 속도를 받음
    B속도를 (임시속도) 로 정하기            ③ B가 저장된 A속도를 받음 ← 교환 완성

[공B · 흰 공]
시작하기 버튼을 클릭했을 때
B속도를 0 로 정하기
계속 반복하기:
  x 좌표를 (B속도) 만큼 바꾸기
교육 단순화 — 같은 질량 가정

본문 🧩 구현은 같은 크기(질량) 두 공의 속도 스왑(완전 탄성, 등질량)으로 단순화했다. 질량이 다른 일반 충돌은 ⭐ 일반 공식(v1'=((m1−m2)v1+2m2v2)/(m1+m2))으로 분리(난이도 단계화). 데모는 일반 공식을 써서 무게별 차이를 보여주지만, 따라 만들기는 등질량 스왑으로 쉽게 시작한다.

물리 정확성 (authoring-editing-guide A-3)

설계 결정

주제 — 물리 계속(2순위) vs 점수·게임 규칙

리뷰 로드맵의 7편은 「점수·게임 규칙」이었으나, 그건 물리가 아니라 게임 메커닉이라 .goals 🔬물리 칼럼이 어색해진다. 사용자가 시리즈 정체성 유지를 위해 2순위 「운동량·충돌」(물리)을 선택. 「점수·게임 규칙」은 index "8편 후보"로 이월.

색 — 와인·버건디(--c7)

1~6편이 빨강·파랑·보라·주황·청록·마젠타를 써서, 가장 덜 겹치는 깊은 와인 #9a3b52 선택. .c-wine .step-no만 공유 CSS에 추가, 나머지 액센트는 페이지 전용 <style>의 var(--c7).

마지막 편 CTA + 후보 블록 이월

7편이 새 마지막 편 → CTA는 index 복귀, 6편은 finale 문구를 "다음 편" 톤으로 demote. index "7편 후보(점수·게임 규칙)"는 7편이 운동량·충돌로 채워졌으므로 "8편 후보(점수·게임 규칙 / 에너지 / 경사면)"로 이월.
