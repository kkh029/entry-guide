6편 「힘과 질량」 추가

작성일 2026-05-31 대상 docs/entry-game-6-force.html PR #27

힘·질량과 뉴턴 제2법칙 직관(가속도 = 힘 ÷ 질량)을 가르치는 6편을 시리즈 새 마지막 편으로 추가하고, 질량 슬라이더로 같은 힘에도 가속도가 달라지는 것을 체감하는 "상자 밀기" 캔버스 데모를 만들었다.

TL;DR
  • next-steps.md "후속 물리 주제 편 추가" 1순위이자 review_20260529.md가 6편으로 명시 권장한 「힘과 질량 — 무거운 상자 밀기 퍼즐」을 신규 추가.
  • 핵심 개념: 같은 힘이라도 질량이 크면 가속도가 작다(a = F ÷ m). 2·3편의 가속·마찰을 "힘÷질량"으로 한 단계 확장.
  • 엔트리 핵심 블록 = 상자속도에 (미는힘 ÷ 질량) 만큼 더하기 — 계산 카테고리 나누기(÷) reporter 중첩(5편 ×와 동일 경로, 텍스트 폴백 불필요) + 마찰 ×0.9.
  • 인터랙티브 데모: 질량 슬라이더(1~10)로 같은 힘(FORCE=0.6)의 가속도 변화를 체감 — 질량 1→가속도 0.60, 10→0.06.
  • 새 액센트 --c6(장미빛 마젠타 #b05a8e) + .c-magenta. 6편이 새 마지막 편 → 5편 CTA를 6편으로 잇고, index "6편 후보"는 "7편 후보(점수·게임 규칙)"로 전환.

개요

기존 시리즈는 1편(이동·충돌) → 2편(속도·가속도) → 3편(관성·마찰) → 4편(포물선·발사) → 5편(튕김·탄성)으로, 5편이 마지막이었다. 외부 리뷰(review_20260529.md)의 "추가로 다루면 좋은 주제 제안"에서 1순위가 「힘과 질량 — 무거운 상자 밀기 게임」이었고, 같은 리뷰가 6편 제목으로 이를 명시 권장했다. 속도·가속도·마찰까지 온 흐름에 "같은 힘이라도 질량이 크면 천천히 움직인다"는 뉴턴 제2법칙 직관을 더하기 좋은 자리였다.

제목·범위·디자인은 사용자 결정으로 확정: 제목 「힘과 질량 — 무거운 상자를 밀어 퍼즐을 풀자!」, 인터랙티브 캔버스 데모 포함, 액센트 --c6 = #b05a8e(장미빛 마젠타).

이슈 — 채워지지 않은 빈틈

시리즈 어디에서도 힘·질량·뉴턴 제2법칙을 다루지 않았다. 캐릭터는 키를 누르면 무엇을 밀든 똑같은 빠르기로 움직였고, "왜 무거운 물체는 같은 힘에 더 느린가"를 설명·구현할 자리가 없었다.

해결 — 6편 신규 + 시리즈 배선
  • 개념↔구현 분리 구조(3편 파일럿 표준): .goals 🔬/🧩 → 🔬힘·질량 → 🔬가속도=힘÷질량 → 🧩엔트리 만들기 → 🔧안 될 때 확인 → ⭐도전 → 🔍코드 찾기 → ✅마무리.
  • 상자 밀기 데모: 질량 슬라이더로 같은 힘에도 가속도가 달라지는 것을 체감. 5편 .try-demo 마크업 + self-contained IIFE/requestAnimationFrame 패턴 재사용.
  • 3·5편 마찰과 명시적 연결: ×0.9 = "속도에 1보다 작은 수 곱하기"의 같은 아이디어(3편 ×0.85, 5편 ×0.7).
  • 시리즈 배선: 5편 종료 CTA를 6편 브리지로 교체(완주 문구 제거), index에 ep-6 카드·6스텝 path 추가, "6편 후보"→"7편 후보(점수·게임 규칙)" 전환, 4편 CTA "마지막 한 편"→"다음 편" 정정.

데모 물리

캔버스 좌표는 수평(x) 1차원만 쓴다(좌표 부호 혼동 회피). 일정한 힘 FORCE를 미는 동안 매 프레임 vel += FORCE / mass로 가속하고, 마찰(vel *= friction)로 감속한 뒤 위치를 적분한다. 질량 슬라이더가 크면 같은 FORCE에도 가속도(FORCE/mass)가 작아져 상자가 눈에 띄게 느려진다 — 이게 a=F/m을 손으로 체감하는 핵심이다.

상수/변수값의미
FORCE0.6 (고정)항상 같은 미는 힘
mass슬라이더 1~10 (기본 4)상자 질량 — 클수록 무거움
friction0.92매 프레임 곱하는 마찰(3·5편과 같은 원리)
가속도 HUDFORCE / mass질량 1→0.60, 4→0.15, 10→0.06
성공 판정x + boxSize ≥ goalX상자 오른쪽 끝이 🎯 버튼 중앙에 닿으면 "퍼즐 성공!"
 
function loop(){
  if (!running) return;
  if (pushing && !success) vel += FORCE / mass;   // 가속 = 같은 힘 ÷ 질량 (핵심)
  vel *= friction;                                 // 마찰 (3·5편과 같은 원리)
  x += vel;                                         // 위치 적분
  if (x < 6){ x = 6; vel = 0; }
  if (x + boxSize > W - 6){ x = W - 6 - boxSize; vel = 0; }
  if (!success && x + boxSize >= goalX){ success = true; pushing = false; }  // 버튼 도착
  render();
  if (pushing || Math.abs(vel) > 0.05) requestAnimationFrame(loop);
  else { running = false; updateBtn(); }
}
HUD 설계 — "가속도(힘÷질량)"를 보여줌

우상단 HUD는 속도가 아니라 가속도 = FORCE/mass를 표시한다. 학습 목표(같은 힘 + 큰 질량 = 작은 가속도)가 슬라이더를 움직이는 즉시 숫자로 드러나게 하기 위해서다(밀기 전에도 갱신).

엔트리 블록 표현

핵심 가속 줄은 계산 카테고리 나누기(÷) 블록을 자료 "더하기" 블록에 중첩해 표현한다(5편 곱하기 ×와 동일 패턴).

시작하기 버튼을 클릭했을 때
질량을 4 로 정하기                         ← 클수록 무거운 상자
미는힘을 5 로 정하기                        ← 항상 같은 힘
상자속도를 0 로 정하기
계속 반복하기:
  만일 〈오른쪽 화살표 키가 눌러져 있는가?〉 이라면:
    상자속도에 (미는힘 ÷ 질량) 만큼 더하기   ← 가속 = 힘 ÷ 질량
  상자속도를 (상자속도 × 0.9) 로 정하기       ← 마찰
  x 좌표를 (상자속도) 만큼 바꾸기
렌더러 — ÷ reporter 중첩 정상 동작 (텍스트 폴백 불필요)

B-mini 렌더러(renderer.js)의 measureContent/renderContent case 'block'이 자식 reporter를 재귀 렌더한다. 5편의 × -0.7과 똑같은 경로라 미는힘 ÷ 질량 CALC reporter도 흰 칸 안에 정상 중첩된다(5편 정착 부등식 같은 비교식 중첩만 텍스트 폴백 대상). 프리뷰에서 SVG 렌더 확인.

설계 결정

주제 — 1순위 그대로

리뷰의 추가 주제 1~8순위 중 1순위(힘·질량)를 채택. 속도·가속도·마찰 다음의 자연스러운 물리 흐름이고, 리뷰가 6편 제목으로 명시 권장했다. 2순위 운동량·충돌(공끼리)은 보류.

색 — 장미빛 마젠타(--c6)

1~5편이 빨강·파랑·보라·주황·청록을 이미 써서, 편 구분이 또렷하도록 기존 5색과 가장 덜 겹치는 #b05a8e를 선택. .c-magenta .step-no만 추가하고 나머지 액센트(CTA·데모 버튼·박스 보더)는 페이지 전용 <style>에서 var(--c6)로 적용(5편식 단색 표준).

마지막 편 CTA + 6편 후보 전환

6편이 새 마지막 편이므로 CTA는 목차(index) 복귀(new-page-guide 권고). 5편은 더 이상 finale가 아니므로 "시리즈 완주" 문구를 "다음 편" 톤으로 낮췄다. index "6편 후보"는 리뷰의 7편 로드맵(점수·게임 규칙)에 맞춰 "7편 후보"로 전환.
